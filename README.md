## Соглашения

### Именование файлов
- Формат: `[YYYY-MM-DD]-[тема]-[тип].расширение`
- Разделитель — дефис, нижний регистр, без пробелов
- Тип: `-vk.md` / `-tg.md` / `-vk.txt` / `-tg.txt` / `-prompt.md` /
  `-image.png`
- Пример: `2026-07-05-northern-ireland-teachers-vk.md`

### Поток публикации
1. Черновик пишется в `content/posts/drafts/<date>-<topic>-<platform>.md`
2. Параллельно — промпт в `<date>-<topic>-prompt.md`
3. Картинка кладётся в `content/assets/<YYYY-MM>/<date>_<topic>/`
4. Публикация в ТГ: `python scripts/publish/tg_poster.py <draft>`
5. Публикация в ВК: ручная (пока нет user_token)
6. После публикации — перенести файл в `content/posts/published/` и
   добавить строку в `content/logs/published.md`
7. **Сразу после публикации** (ручной или авто) — обновить запись в
   `content/logs/published.md`: статус «✅ Опубликовано», способ
   (автопостер / вручную), `message_id` (ТГ) или `post_id` (ВК) и
   ссылку. Без отметки публикация считается неподтверждённой.
8. **Технические публикации** (проверка токена, smoke-test постинга,
   эксперимент) тоже фиксируются — в отдельном блоке «технические
   тесты» с префиксом `T` в номере (`T1`, `T2`, ...). Скрытые
   посты помечаются явно. Удалённые — `удалено` со ссылкой на
   archive.

### Конфигурация и секреты
- **Все ключи** — в `.env` или `.env.local`. В коде — только через
  `python-dotenv`. Никаких хардкод-токенов.
- Шаблон без секретов — `.env.example` (в репо). Скопируй в `.env` и
  заполни реальными значениями.
- VK-приложения (см. `docs/archive/`):
  - `54646486` **NeuroUchitel Post** — рабочее для автопостинга в ВК
  - `54664289` **NeuroUchitel Auth** — для OAuth в `neuro-teacher-pro`
- Telegram: бот `@neuro_uchitel_content_bot`, канал `@neuro_uchitel`

### Что НЕ коммитим
- `.env`, `.env.local` (секреты)
- `content/posts/drafts/*.txt` (рабочие артефакты для копипаста в
  редактор; хранятся локально до публикации)
- Вложенные `.git/` из архивных прототипов (правило `**/.git/`)
