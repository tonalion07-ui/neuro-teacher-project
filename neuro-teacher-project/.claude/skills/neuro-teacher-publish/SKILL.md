# neuro-teacher-publish

Оркестратор полного цикла публикации одной связки постов (ВК + ТГ) для канала
«Нейроучитель: AI в школе». Сам не публикует — делегирует существующим
инструментам и ведёт журнал.

## Когда вызывать

Пользователь говорит одно из:
- «Опубликуй [дату]»
- «Запости связку от [дата]»
- «Публикуем [тему]»
- «Пост от [дата] в ВК и ТГ»
- «Опубликуй [имя файла]»

Не вызывать для:
- генерации текстов (это `content-creator` / `neuro-teacher-to-social`).
- только ТГ-публикации (делегируй `tg-poster-skill` напрямую).
- только ВК-публикации без ТГ-пары (можно, но это исключение — см. §«Режимы»).

## Вход

| Параметр | Описание | Обязательно |
|---|---|---|
| `draft_date` | Дата публикации в `YYYY-MM-DD` | да |
| `topic_slug` | Ключ темы в имени файлов (например, `selskiy-uchitel`) | да, если в `drafts/` несколько тем на эту дату |
| `vk_mode` | `auto` (вызвать `vk_poster.py`, когда OAuth готов) \| `manual` (выдать текст + image для ручной вставки). По умолчанию `manual` | нет |
| `force_no_review` | Пропустить шаг 3-агентного ревью. По умолчанию `false` | нет |

Скилл **сам** ищет 3 файла в `content/posts/drafts/`:

```
{date}-{slug}-vk.md
{date}-{slug}-tg.md
{date}-{slug}-image-prompt.md
```

Картинка (если была сгенерирована) ищется в
`content/assets/{YYYY-MM}/{date}-{slug}/{date}-{slug}.png`.

## Режимы ВК

По умолчанию `manual`, потому что OAuth-флоу через `vk_poster.py` **пока
не настроен** (см. `docs/2026-07-08-vk-oauth-migration-plan.md`).

| Режим | Что делаем |
|---|---|
| `manual` (по умолчанию) | Скилл готовит комплект: очищенный от Markdown текст поста, путь к картинке, hashтеги, ссылку на ТГ-тизер, и **выдаёт пользователю** в чате. Публикацию пользователь делает руками через wall.post в интерфейсе ВК |
| `auto` | Вызывает `python scripts/publish/vk_poster.py <vk.md> [image]` — **только** если пользователь явно подтвердил, что OAuth готов. Без подтверждения `auto` отключается в `manual` с предупреждением |

Community-токен (`VK_COMMUNITY_TOKEN` / `VK_ACCESS_TOKEN`) годится только
для текстовых постов. Для постов с картинкой нужен user-token через OAuth 2.1.
Это правило 11.07.2026, не нарушать.

## Алгоритм

### Шаг 1. Валидация входа

Перед стартом проверить:

1. Все 3 файла существуют. Если хоть один отсутствует — **СТОП**, спросить.
2. `.env` существует в `neuro-teacher-project/.env`. Без него ничего не запустится.
3. Токены для ТГ: `TELEGRAM_BOT_TOKEN` и `TELEGRAM_CHAT_ID`.
4. Токены для ВК: `VK_USER_ACCESS_TOKEN` (приоритет, для `photos.*`) или
   `VK_ACCESS_TOKEN` (fallback, только text-only). См. `CLAUDE.md` §3.3.
5. В `content/logs/published.md` и `content-plan/published-log.md` нет
   записи об этой дате+теме (не дублируем).
6. Длина ТГ-тела (между `---` в `*-tg.md`) после strip markdown
   ≤ 1024 (для sendPhoto) или ≤ 4096 (для sendMessage).

Если что-то не так — **СТОП**, спросить пользователя.

### Шаг 2. 3-агентное ревью (обязательно, если ещё не пройдено)

Перед публикацией прогнать тексты через 3 субагента **параллельно** через
`Agent` tool с `subagent_type` (правило `CLAUDE.md` §3.1):

| Агент | Проверяет | Артефакт |
|---|---|---|
| `content-reviewer` | Стиль, тон, хэштеги, оформление | список правок |
| `fact-checker` | Фактология, 152-ФЗ, ФГОС, CTA | список правок |
| `style-reviewer` | Стилистика, голос, Markdown-артефакты | список правок |

После 3 проверок — единый список правок → один `Edit`-проход во все 3 файла
→ **спросить пользователя «OK» на публикацию** → переход к шагу 3.

Пропускается только если `force_no_review=true` (экстренная перепубликация).

### Шаг 3. Публикация в Telegram

Делегировать `tg-poster-skill` с параметрами:

| Параметр | Значение |
|---|---|
| `draft_path` | `content/posts/drafts/{date}-{slug}-tg.md` |
| `image_path` | `content/assets/{YYYY-MM}/{date}-{slug}/{date}-{slug}.png` (если есть) |
| `publish_date` | `{date}` |

`tg-poster-skill` сам:
1. Сделает strip markdown и двойную проверку длины.
2. Запустит `python scripts/publish/tg_poster.py`.
3. Вернёт `message_id` и ссылку `https://t.me/neuro_uchitel/{id}`.

Если `message_id` не получен — **СТОП**, ВК не запускаем.

### Шаг 4. Публикация в ВКонтакте

#### Режим `manual` (по умолчанию)

Подготовить и **выдать пользователю в чат**:

1. **Текст поста** — берём тело из `*-vk.md` между `---`, очищаем
   от Markdown-артефактов (правило `CLAUDE.md` §3.2 и `style-reviewer` §7).
   Запрещены: `##`, `###`, `**`, `__`, `` ` ``, `>`, `---`.
   Заменяем на эмодзи-маркеры 🔹 ✅ 💡 или обычный текст.
2. **Путь к картинке** — относительный путь в `content/assets/...` (для скачивания).
3. **Хэштеги** — 3–7 штук, в конце поста, из списка:
   `#Нейроучитель #AIвшколе #учительсAI #нейросети #образование`
4. **Ссылка на ТГ-тизер** — подставить первой строкой для перелинковки.
5. **Краткая инструкция**: «Открой wall.post в управлении сообществом,
   вставь текст, прикрепи картинку, опубликуй. Скинь сюда ссылку
   `https://vk.com/wall-{group_id}_{post_id}` после публикации».

Дождаться от пользователя ссылки на опубликованный пост. Без ссылки
шаг 5 (журнал) не выполняется.

#### Режим `auto` (только если пользователь подтвердил готовность OAuth)

```bash
python scripts/publish/vk_poster.py content/posts/drafts/{date}-{slug}-vk.md [image_path]
```

`vk_poster.py` имеет пре-чек `check_vk_markdown()` — если сработает,
правки вносим в `*-vk.md` ДО повторного запуска (или добавляем `--force`
для экстренной отправки, но это исключение).

Ожидаемый вывод: `[OK] Пост опубликован. post_id: <id>`.

### Шаг 5. Обновить журналы (обязательно)

Согласно `CLAUDE.md` §3.4 и `content-plan/published-log.md` — после
успешной публикации обновить **три** файла:

| Файл | Что меняем |
|---|---|
| `content/logs/published.md` | Добавить строку с датой, темой, площадкой, ссылками. Для ВК: `https://vk.com/wall-{group_id}_{post_id}`. Для ТГ: `message_id` + URL |
| `content-plan/published-log.md` | То же, с матрицей метрик (охват/лайки/комменты/пересылки — заполняется через 24ч) |
| `content-plan/content-plan-month.md` | Статус темы ⬜ → ✅ |

И перенести черновики:
```bash
git mv content/posts/drafts/{date}-{slug}-vk.md  content/posts/published/
git mv content/posts/drafts/{date}-{slug}-tg.md  content/posts/published/
git mv content/posts/drafts/{date}-{slug}-image-prompt.md content/posts/published/
```

Удалить `.txt`-дубликаты из `drafts/`, если были.

### Шаг 6. Возврат пользователю

Сообщить:
- 2 ссылки (ВК + ТГ).
- Краткое «что сделано» (1–2 строки).
- Напоминание про аналитику через 24ч: «Завтра снять статистику,
  записать в `content/logs/analytics-log.md`».

## Откат

| Площадка | Как откатить | Когда можно |
|---|---|---|
| Telegram | `editMessageText` → «[Удалено автором]» | Всегда. `deleteMessage` не использовать (лимит 48ч) |
| ВК | `wall.delete` через API | В первые 24ч. После — только редактирование текста |
| Журналы | Заменить ✅ на 🟥 Удалено в обоих `published.md` | — |
| Файлы | `git mv` обратно в `drafts/` | — |

## Чек-лист безопасности (перед стартом)

- [ ] Все 3 файла черновика существуют
- [ ] `.env` содержит `TELEGRAM_BOT_TOKEN`, `TELEGRAM_CHAT_ID`
- [ ] `.env` содержит `VK_USER_ACCESS_TOKEN` ИЛИ пользователь явно выбрал `vk_mode=manual`
- [ ] Ревью 3 агентов пройдено ИЛИ `force_no_review=true`
- [ ] Длина ТГ-тела ≤ 1024 (sendPhoto) / ≤ 4096 (sendMessage)
- [ ] В журналах нет дубля этой даты+темы
- [ ] Пользователь дал явное «OK» на публикацию (мутирующая операция)

## Известные ограничения

1. **OAuth-флоу ВК не настроен** — по умолчанию `vk_mode=manual`.
   План в `neuro-teacher-pro/docs/2026-07-08-vk-oauth-migration-plan.md`
   (файл в параллельном проекте, не читал).
2. **Community-токен НЕ годится для `photos.*`** (error 27) — для постов
   с картинкой только user-token через OAuth. Правило 11.07.2026.
3. **Markdown-артефакты в ВК** — ВК не рендерит `##`, `**`, `__`,
   `` ` ``, `>`, `---`. В `manual`-режиме скилл сам чистит и выдаёт
   готовый текст. В `auto` — срабатывает `check_vk_markdown()`.
4. **Хэштеги** в ТГ через 🔹 (не Markdown), в ВК — `#`.
5. **Цитаты и факты** — перед публикацией фактчек обязателен.
   AB 2876 (Калифорния), учебник литературы (Чацкий, 15.07) — задачи
   из бэклога.
6. **152-ФЗ, ФГОС** — проверяет `fact-checker`.
