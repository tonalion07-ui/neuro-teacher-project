## Правила использования

1. **Свежесть.** Каждая разведка — отдельный файл с датой в имени: `news-YYYY-MM-DD.json`.
2. **Цитирование.** Любая цифра/факт из поста → ссылка на источник в банке или прямой URL.
3. **Use-before дата.** Источники с дедлайном (призывы, конкурсы) — помечать `use_before` в JSON.
4. **Непроверенные URL.** Помечать `verified: false` в JSON, не использовать без ручной проверки.
5. **РФ-контекст — приоритет.** Минимум 1 из 3 постов должен опираться на РФ-источник (аудитория русскоязычная).
6. **Ротация типов.** Не повторять один тип 2 недели подряд (см. `topics-rotation.md`).
7. **Обновление.** Раз в неделю — новая разведка, старые не удаляем (история решений).

## Как пользоваться

1. При подготовке поста — открыть свежий JSON, выбрать 1–2 источника нужного типа.
2. Проверить, не использовался ли этот источник в последних 2 постах.
3. Свериться с планом (`content/plan/july.md`, `content/plan/monthly.md`) — тема должна бить в рубрику.
4. После публикации — обновить `content/logs/published.md`, указав ID источника из банка.
