## 1. Контекст и точка старта

`neuro-teacher-project` — действующий контент-завод по теме «AI в школе» для учителей. Площадки: ВК-группа «Нейроучитель: AI в школе» + Telegram `@neuro_uchitel`. С 22.06.2026 опубликовано 22 связки (ВК + ТГ), есть 3-агентный ревью-конвейер (content-reviewer / fact-checker / style-reviewer), ТГ-автопостер стабилен, ВК-автопостер переключён на community-token (text-only + ручная вставка картинки).

**Цель месяца:** стабилизировать конвейер на полной сетке Пн/Ср/Пт, выйти на 50 новых подписчиков ТГ в неделю и средний охват 30 на пост в ВК, опубликовать рабочий MVP-пакет в публичном GitHub-репо и сдать ДЗ модуля 4.

**GitHub-репо для сдачи:** `https://github.com/tonalion07-ui/neuro-teacher-project` (публичное с 14.07.2026; новое отдельное репо `neuro-teacher-pro` не создаём — весь код и артефакты идут в существующее).

## 2. Сценарий пользователя

**Кто:** школьный учитель-предметник, 35–60 лет, преимущественно мужчины. Ведёт 1–3 предмета, 5–11 класс, город или сельская школа. Уровень работы с ИИ — нулевой или «пробовал ChatGPT, не разобрался». Подключение к интернету — стабильное или умеренно нестабильное, время — фрагментами по 20–40 минут между уроками.

**Боли:**
- Тратит 6–10 часов в неделю на рутину (проверка тетрадей, отчёты, планы, справки).
- Тонет в потоке новостей про AI в образовании — нет фильтра «что мне, в мой предмет».
- Не умеет формулировать задачи для нейросети: пишет «сделай конспект» — получает мусор.
- Скептически относится к «западным» ИИ: «а наши-то есть? доверять можно?».
- Не хочет тратить вечера на «освоение новой технологии» — нужны готовые решения, копируй и подставляй.

**Ценность, которую получает:**
- Готовые промпт-шаблоны и сценарии под предмет и ситуацию.
- 3 поста в неделю в удобном формате: ВК — длинная версия с разбором, ТГ — тизер со ссылкой.
- Фактчек и адаптация под РФ (GigaChat, YandexGPT, ChatGPT — с пометкой про VPN).
- Можно взять в работу сразу: скопировать промпт, подставить свой предмет/класс, не ломать голову.

**Путь пользователя (3 шага):**
1. **Обнаружение** — находит пост в ВК или ТГ (через ленту, рекомендации, репост коллеги).
2. **Применение** — копирует промпт/шаблон, подставляет свой предмет, класс, тему.
3. **Возврат** — ставит лайк/реакцию, через неделю видит следующий пост в той же рубрике и возвращается.

**Анти-сценарий (что мы НЕ пытаемся):**
- Не превращаем учителя в «продвинутого пользователя AI».
- Не учим писать промпты с нуля — даём готовые конструкции.
- Не гонимся за хайпом про новые модели — фокус на стабильной тройке (GigaChat / YandexGPT / ChatGPT).

## 3. Метрики успеха

### 3.1. Продуктовые (что измеряем)

| Метрика | Что значит | Источник | Целевое значение к 13.08 |
|---|---|---|---|
| Публикаций в неделю | Связки ВК + ТГ в сетке Пн/Ср/Пт | `content/logs/published.md` | 3/нед, без пропусков 4 недели подряд |
| Стабильность сетки | Дней без пропусков подряд | журнал | 14+ |
| Вовлечение ВК (среднее на пост) | Лайки + комменты + пересылки / пост | ручной замер через 24 ч | ≥ 30 |
| Рост ТГ-подписчиков | Новых / неделю | ручной замер | ≥ 50/нед |
| Прохождение 3-агентного ревью | % постов, прошедших правки | `content/posts/review/` | 100% (правило `CLAUDE.md` §3.1) |
| Доля автопубликаций | Постов, ушедших через `vk_poster.py` / `tg_poster.py` без ручной вставки | журнал | ТГ 100%, ВК 100% (text-only + ручная вставка фото через UI) |

### 3.2. Технические (что строим)

| Метрика | Целевое значение |
|---|---|
| E2E-цикл оркестратора | Команда → готовая связка в журнале ≤ 5 минут, 0 ручных шагов кроме «OK» |
| Время от черновика до публикации | ≤ 30 минут (3-агентное ревью + правки + публикация) |
| Скилл `neuro-teacher-publish` | Развёртывается в новом окружении по README за ≤ 10 минут |
| README репо | Проходит «тест друга» — посторонний человек понимает проект за 3 минуты |
| Скриншоты в репо | 6–8 штук, покрывают весь цикл (черновик → ревью → дифф → ТГ → ВК → журнал) |

### 3.3. Метрики, которые НЕ измеряем (и почему)

- **Подписчики ВК** — ВК-группа не наша, метрика вне зоны контроля.
- **Конверсия в продажу** — продаж нет, проект контентный, не коммерческий.
- **Время генерации картинки** — упираемся в Midjourney/Fal.ai, не оптимизируем.
- **Среднее время сессии в ВК** — инструментарий ВК-аналитики ограничен, шумно.
