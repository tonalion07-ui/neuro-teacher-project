## 3. Детализация по пунктам

### 3.1. Контент-блоки: примеры работ (п. 1)

- **Обоснование:** раздел «Примеры работ» в `index.html` сейчас — три карточки-плейсхолдера с подписями «Методичка / Конспект / Карточки». Это анти-доверие: посетитель видит, что «примеров» нет, и уходит.
- **Реализация:** подготовить 3 изображения (например, скриншоты из реальных PDF/документов) в формате WebP, положить в `site/assets/sample-1.webp`, `sample-2.webp`, `sample-3.webp`. Разметка в `index.html` (`.samples` → `.sample`) уже есть — менять её не нужно, только заменить `<div class="sample__img">…</div>` на `<img>`.
- **Сложность:** низкая. Сложность — в подготовке самих макетов, не в коде.
- **Приоритет:** сначала.

### 3.2. Скачиваемые материалы — `price.pdf` (п. 2)

- **Обоснование:** CTA «Скачать прайс в PDF» в Hero и блоке CTA — оба ведут на несуществующий файл. Это битая ссылка прямо в основной точке конверсии.
- **Реализация:** подготовить PDF (Google Docs / Canva → экспорт PDF), положить в `site/assets/price.pdf`. Имя файла соответствует канону нейминга (`assets/price.pdf` — внутривенный служебный путь, не пользовательский, поэтому допустимо).
- **Сложность:** низкая.
- **Приоритет:** сначала.

### 3.3. Open Graph, Twitter Card, favicon (п. 3)

- **Обоснование:** `<meta property="og:image">` уже прописан (`assets/og-image.png`), но файла нет. Без OG-image шеринг в Telegram, ВК, WhatsApp выглядит как пустая карточка. Без `favicon.ico` — иконка вкладки в виде глобуса браузера.
- **Реализация:**
  - `assets/og-image.png` — 1200×630 px, ключевая фраза + логотип + подзаголовок «Для педагогов».
  - `assets/favicon.ico` — 32×32 (и/или `.svg` + `<link rel="icon" type="image/svg+xml">`).
  - Дополнить `<head>`: `<meta property="og:url">`, `<meta property="og:locale" content="ru_RU">`, `<meta property="og:site_name" content="Нейроучитель">`, `<meta name="twitter:card" content="summary_large_image">`.
- **Сложность:** низкая.
- **Приоритет:** сначала.

### 3.4. SEO-базис: sitemap, robots, мета-теги (п. 4)

- **Обоснование:** `WebFetch` на `/sitemap.xml` вернул 404. Поиск Яндекса/Google лендинг не индексирует, либо индексирует выборочно. Для образовательного проекта в РФ — Яндекс приоритетнее Google, но sitemap нужен обоим.
- **Реализация:**
  - `site/sitemap.xml` — ссылки на `/`, `/privacy.html`, `/oauth-callback.html` (если индексируется).
  - `site/robots.txt` — `User-agent: *`, `Allow: /`, `Sitemap: https://www.neuro-teacher.pro/sitemap.xml`.
  - Дополнить `<head>` пункта 3.3.
- **Сложность:** низкая.
- **Приоритет:** сначала.

### 3.5. Аналитика (п. 5)

- **Обоснование:** без счётчика невозможно узнать, откуда приходят посетители (Telegram, ВК, поиск, прямые заходы) и какие секции работают, а какие — нет. Это блокирует любое дальнейшее развитие на основе данных.
- **Реализация:**
  - **Яндекс.Метрика** (приоритет для РФ): счётчик + вебвизор + карта кликов. Требует cookie-баннер, если используется стандартная версия, либо «безcookie»-режим (`ym(…,'requireCookieConsent')`).
  - **Plausible** (облако, без cookie-баннера): одна строка `<script defer data-domain="…">`. Проще в настройке, дешевле визуально.
  - **Комбинация:** Plausible как основной + Метрика для углублённой аналитики.
- **Сложность:** низкая.
- **Приоритет:** сначала.

### 3.6. Форма заявки (п. 6)

- **Обоснование:** единственный способ связаться с владельцем — Telegram/WhatsApp/email. Учитель, у которого нет Telegram, отсеивается. Кроме того, при переписке в мессенджере теряется структура: предмет, класс, тема, формат (разовый материал / серия / обучение). Форма фиксирует эту структуру.
- **Реализация — вариант A (без бэкенда, на замену Formspree):**
  - Сервис: [Formspree](https://formspree.io) (бесплатный тариф 50 заявок/мес) или [Getform](https://getform.io).
  - HTML-форма `action="https://formspree.io/f/{id}" method="POST"`, поля: имя, email, предмет, класс, тема, формат, сообщение.
  - Заявки падают на email `vtab3@yandex.ru`.
- **Реализация — вариант B (нативно, рекомендуемый):**
  - Vercel Serverless Function `site/api/lead.js` (Node 20).
  - Принимает `POST { name, email, subject, class, topic, format, message }`.
  - Шлёт уведомление в Telegram-бот (Bot API `sendMessage`, токен в env) **и** на email через [Resend](https://resend.com) (бесплатный тариф 3000 писем/мес).
  - HTML-форма на странице — обычная, `action="/api/lead"`.
- **Секреты:** `TELEGRAM_BOT_TOKEN`, `TELEGRAM_ADMIN_CHAT_ID`, `RESEND_API_KEY` — все в Vercel env.
- **Сложность:** низкая–средняя.
- **Приоритет:** сначала (вариант B, потому что контролируем данные сами, без третьих сторон).

### 3.7. Telegram-бот (п. 7)

- **Обоснование:** сейчас ссылка «Написать в Telegram» — это просто ссылка на `@neuro_uchitel`. Бот может: (а) автоматически отвечать на `/start` приветствием и кнопкой «Заказать методичку», (б) принимать тему/предмет/класс пошагово, (в) дублировать заявку с сайта админу в тот же чат.
- **Реализация:**
  - Vercel Serverless Function `site/api/tg-webhook.js` (Node 20) — обрабатывает `update` от Telegram.
  - Регистрация webhook: `https://api.telegram.org/bot{TOKEN}/setWebhook?url=https://www.neuro-teacher.pro/api/tg-webhook`.
  - Команды: `/start` (приветствие + меню), `/demo` (отправляет превью PDF), `/subject` (запускает диалог выбора предмета).
  - Состояния диалога — в Upstash Redis (бесплатный тариф 10 000 команд/день) или Vercel KV. Без хранилища бот не сможет помнить, на каком шаге диалога пользователь.
- **Сложность:** средняя.
- **Приоритет:** потом (после формы, потому что бот без формы — просто «привет, как дела?»).

### 3.8. Кэширование на Vercel (п. 8)

- **Обоснование:** лендинг статический, но без `Cache-Control` каждый запрос уходит на Vercel Edge. Когда появятся `sample-1/2/3.webp`, `og-image.png`, `price.pdf` — без кэширования они будут качаться каждым посетителем заново.
- **Реализация:** обновить `site/vercel.json`:

```json
{
  "functions": {
    "api/vk-token.js": { "maxDuration": 10 },
    "api/lead.js": { "maxDuration": 10 },
    "api/tg-webhook.js": { "maxDuration": 10 }
  },
  "headers": [
    {
      "source": "/assets/(.*)",
      "headers": [
        { "key": "Cache-Control", "value": "public, max-age=31536000, immutable" }
      ]
    },
    {
      "source": "/(.*)",
      "headers": [
        { "key": "Cache-Control", "value": "public, max-age=3600, s-maxage=86400" }
      ]
    }
  ]
}
```

- **Сложность:** низкая.
- **Приоритет:** сначала (один раз настроить — работает всегда).

### 3.9. Изображения → WebP/AVIF (п. 9)

- **Обоснование:** когда появятся реальные ассеты (sample-превью, og-image), они по умолчанию будут PNG/JPEG. На мобильном это лишние сотни килобайт и низкий PageSpeed.
- **Реализация:** конвертация в WebP (или AVIF, если поддержка родительских браузеров не критична), `<picture>` с `srcset`, явные `width`/`height` против CLS, `loading="lazy"` для превью ниже первого экрана.
- **Сложность:** низкая.
- **Приоритет:** сначала, но строго в связке с пунктами 1 и 3.

### 3.10. PWA / офлайн (п. 10)

- **Обоснование:** для учителя с нестабильным интернетом в школе офлайн-методичка теоретически полезна, но: (а) PWA не покрывает сценарий «я хочу, чтобы страница работала офлайн», потому что лендинг — это воронка, а не контент; (б) материалы лучше отдавать как PDF (открывается в любом ридере офлайн), чем как PWA.
- **Решение:** не нужно.
- **Сложность:** средняя (если бы делали).
- **Приоритет:** не обязательно.

### 3.11. Мультиязычность (п. 11)

- **Обоснование:** аудитория — педагоги РФ, говорящие по-русски. Английская версия не нужна. Белорусская/казахская — преждевременны (нет аудитории).
- **Решение:** не нужно.
- **Сложность:** средняя (если бы делали).
- **Приоритет:** не обязательно.

### 3.12. Авторизация пользователей / личный кабинет (п. 12)

- **Обоснование:** VK ID уже подключён (client_id `54654398`, PKCE, serverless `/api/vk-token`). Но **сценария после входа нет** — пользователь видит чип со своим `user_id` в шапке, и всё. Это авторизация «ради авторизации»: она не решает задачу пользователя, а только добавляет трение.
- **Решение — отложить развитие ЛК до появления контента, ради которого логинятся.** Возможные сценарии: история заявок; библиотека сгенерированных методичек пользователя; сохранённые шаблоны; подписка на обновления.
- **Если ЛК всё-таки нужен — пересмотреть стек:** вместо самописной связки VK ID + Vercel KV использовать **Clerk** (бесплатный тариф до 10 000 MAU, поддерживает VK из коробки, готовые UI-компоненты) или **Supabase Auth** (если БД всё равно понадобится).
- **Сложность:** высокая.
- **Приоритет:** потом (когда появится контент, ради которого логинятся).

### 3.13. База данных (п. 13)

- **Обоснование:** БД под заявки — не нужна, если форма шлёт в Telegram/email. БД под контент — не нужна, лендинг статический. БД под пользователей — нужна **только** вместе с ЛК.
- **Сейчас:** не нужна.
- **Когда понадобится (варианты):**
  - **Supabase** — бесплатный tier 500 МБ Postgres + Auth + Storage + Realtime. Лучший выбор для одиночного разработчика, который хочет «всё из коробки».
  - **Vercel Postgres** — если хочется остаться полностью в экосистеме Vercel, но дороже и без Auth из коробки.
  - **Neon** — serverless Postgres, бесплатный tier 0.5 ГБ, без Auth.
  - **Turso** — если данные гео-распределённые (для РФ-аудитории это не критично).
- **Сложность:** средняя–высокая.
- **Приоритет:** потом (вместе с ЛК).

### 3.14. Бэкенд в целом (п. 14)

- **Обоснование:** сейчас единственная serverless-функция — `/api/vk-token.js`. Этого достаточно для обмена OAuth-кода на токен. Больше бэкенда не нужно, пока не появятся: форма, бот, ЛК.
- **Решение:** добавлять Serverless Functions **точечно**:
  - `/api/lead.js` (форма).
  - `/api/tg-webhook.js` (бот).
  - **Edge Runtime не даст выигрыша** для этих задач — Node 20 достаточен.
- **Сложность:** низкая.
- **Приоритет:** сначала (только под форму и бота).

### 3.15. Юридические мета-теги (п. 15)

- **Обоснование:** `<head>` уже содержит `og:title`, `og:description`, `og:image`, но не содержит `og:locale`, `og:url`, `og:site_name`, `twitter:card`. Эти теги влияют на корректность отображения в Telegram, ВК, Slack, Twitter.
- **Реализация:** дополнить `<head>`:

```html
<meta property="og:url" content="https://www.neuro-teacher.pro/" />
<meta property="og:locale" content="ru_RU" />
<meta property="og:site_name" content="Нейроучитель" />
<meta name="twitter:card" content="summary_large_image" />
```

- **Сложность:** низкая.
- **Приоритет:** сначала.

### 3.16. UX-плейсхолдеры (п. 16)

- **Обоснование:**
  - Hero-карточка с подписью «Демо-превью. Реальный макет появится позже» — **на главном экране** подрывает доверие.
  - Кнопка «Войти через VK ID» с `href="#"` и непонятным сценарием после входа — **снижает конверсию**: пользователь кликает, проходит OAuth, и оказывается на той же странице с чипом `VK user_id: …`. Зачем?
- **Реализация:**
  - **Hero-карточка:** убрать подпись «Демо-превью…» или сделать её визуально неакцентной (`opacity: 0.5`); либо заменить плейсхолдер скриншотом реального интерфейса (когда появится).
  - **Кнопка VK ID:** пока сценария нет — **убрать кнопку** совсем. Когда сценарий появится (ЛК) — вернуть с подписью «Чтобы сохранять методички» или подобной.
- **Сложность:** низкая.
- **Приоритет:** сначала.

### 3.17. Видео-демо (п. 17)

- **Обоснование:** для аудитории «не технарей» 60-секундное видео «как я готовлю конспект за 4 минуты» конвертит лучше, чем текст. Это заменяет сломанную hero-карточку на работающий «продающий» блок.
- **Реализация:** записать 60-сек. видео (экран + голос), загрузить на YouTube или RuTube, встроить в hero-блок через `<iframe>` (RuTube предпочтительнее для РФ-аудитории, нет проблем с блокировками). Альтернатива — `<video>` с `poster` напрямую.
- **Сложность:** низкая (сложность — в съёмке, не в коде).
- **Приоритет:** потом (когда будет готов контент).

## 4. План на ближайшие 2 недели

### Неделя 1 — контент и измеримость

1. Подготовить и положить в `site/assets/`:
   - `og-image.png` (1200×630).
   - `favicon.ico` (или `.svg`).
   - `price.pdf` (прайс с примерами и тарифами).

## 5. Что не трогать на текущем этапе

- **PWA / офлайн-режим** — окупаемость нулевая.
- **Мультиязычность** — аудитория русскоязычная.
- **Своя БД** — пока не нужна, всё уходит в Telegram/email.
- **Полноценный личный кабинет** — нет контента, ради которого логинятся.
- **Переписывание на Next.js** — текущий стек справляется, миграция не оправдана.
- **Сложные CMS-интеграции** — контент лендинга меняется редко, проще править HTML.

## 6. Стратегические развилки (на будущее)

Когда текущий лендинг окупится и появится спрос:

- **Библиотека методичек.** Готовые методички в виде PDF по предметам и классам, каталог с фильтрами. Это потребует БД и, скорее всего, миграции на Next.js (для динамических страниц `/subjects/literature/9/`).
- **Платный доступ.** Сейчас заявлено «Бесплатно для учителей». Если ввести платные тарифы — потребуется платёжная система (ЮKassa, Robokassa, Prodamus). Это история про полноценный ЛК.
- **Партнёрская программа.** Учитель приводит коллегу — получает бонус. Потребует реферальных ссылок и трекинга конверсий.
- **Email-рассылка.** Сейчас email только для связи. Сбор email-базы через форму → регулярная рассылка (через Resend или Unisender). Это отдельный канал привлечения.

Каждая из этих развилок — отдельный проект, не «доработка лендинга».

## 8. Концепции, на которые стоит обратить внимание для роста

- **Vercel Functions vs Edge Runtime** — для текущих задач Node 20 достаточен, Edge даст выигрыш только если появится гео-логика или A/B-тесты на edge.
- **OAuth 2.1 + PKCE** — уже применён в `auth.js`. Хороший пример того, как делать безопасный client-side OAuth без хранения client_secret в браузере.
- **Resend как альтернатива SMTP** — современный API для транзакционных писем, бесплатный тариф 3000/мес, проще, чем настраивать SMTP-relay.
- **Cache-Control: immutable** — для файлов в `/assets/` с хешем в имени (или без — если контент обновляется через редеплой) — браузер не будет перепроверять на сервере, что ускоряет загрузку.
- **Serverless cold start** — на Vercel Hobby до 1–3 секунд на первый запрос после простоя. Для формы это терпимо, для вебхука Telegram — нет (Telegram retry-ит при таймауте). Решение: warm-up через cron или переход на Vercel Pro с `concurrency`.
